# Подключение CRM Bitrix24

Парсер заводит каждый годный лид в Bitrix24 как **лид воронки** — с оценкой,
ориентиром суммы, ссылкой на сообщение в Telegram и полной карточкой в комментарии.
Подключение через **входящий вебхук** — без OAuth, за пару минут.

## 1. Создать входящий вебхук

1. Зайди в свой Bitrix24 под админом.
2. Слева: **Разработчикам** (или «Приложения» → «Разработчикам»).
3. Выбери **Другое → Входящий вебхук** (Inbound webhook).
4. В правах (scope) отметь **CRM** (`crm`).
5. Сохрани. Скопируй URL вида:
   ```
   https://ВАШ-ПОРТАЛ.bitrix24.ru/rest/1/x1y2z3abc4def5gh/
   ```
   (число `1` — ID пользователя, `x1y2...` — секретный код вебхука).

> Этот URL — секрет: кто им владеет, тот пишет в твою CRM. Не коммить его в git.

## 2. Прописать в .env

```
ANTHROPIC_API_KEY=sk-ant-...          # CRM работает поверх LLM-карточки
LLM_ENABLED=true

BITRIX_WEBHOOK_URL=https://ВАШ-ПОРТАЛ.bitrix24.ru/rest/1/x1y2z3abc4def5gh/
CRM_ENABLED=true
CRM_MIN_SCORE=warm                    # в CRM только warm и выше (hot/warm)
```

## 3. Запустить

```
pip install -r requirements.txt
python main.py
```

Теперь поток такой:

```
сообщение → фильтр → LLM-классификация → лид
   ├─ карточка в Telegram-группу (если рейтинг ≥ LLM_MIN_SCORE)
   └─ лид в Bitrix24 (если рейтинг ≥ CRM_MIN_SCORE)
```

## Что попадает в карточку лида Bitrix24

- **Название**: `🔥HOT Лендинг премиум-дизайн — Барбершоп ищет дизайнерский лендинг`
- **Сумма (OPPORTUNITY)**: середина ценовой вилки заказа, в рублях
- **Источник**: канал + ссылка на сообщение (+ @username автора, если открыт)
- **Контакт (NAME/IM)**: имя и ссылка на Telegram автора, если доступны
- **Комментарий**: полная карточка — ниша, тип клиента, бюджет, срок, как отвечать, причины оценки
- **Стадия**: NEW (новый лид) — дальше двигаешь по воронке руками

## Заметки на будущее

- **Дедупликация**: сейчас каждое сообщение = новый лид. Если один человек постит
  в нескольких чатах, будут дубли. Позже можно искать существующий лид по
  Telegram-username (`crm.lead.list`) и не плодить копии.
- **Своя воронка/поля**: можно завести кастомные поля (UF_*) под score, нишу,
  цену — и фильтровать/сортировать воронку прямо по ним. Сейчас всё в стандартных
  полях, чтобы работало без настройки.
- **Сделки вместо лидов**: если работаешь в режиме «сразу сделки», поменяй в
  `core/crm.py` метод `crm.lead.add` на `crm.deal.add` (поля совместимы по смыслу).
